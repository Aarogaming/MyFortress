# MyFortress Long-Term Roadmap (Policy Fortress Program)

Status: Proposed (execution-ready)
Date: 2026-02-19
Scope: `AaroneousAutomationSuite/MyFortress`

## Purpose

Harden MyFortress into the policy and security enforcement layer for the
workspace autonomy program.

## Baseline

Current capabilities in-repo:

1. Policy evaluator and negative-path checks:
   - `scripts/evaluate_policy_gate.py`
   - `scripts/check_policy_gate_negative.py`
2. CP3 readiness and CP4 status pipeline:
   - `scripts/evaluate_cp3_readiness.py`
   - `scripts/run_policy_pipeline.py`
   - `scripts/render_cp4_status.py`
3. Governance documentation and rollout policy:
   - `docs/policy/JANUS_GATE_ENFORCEMENT_ROLLOUT.md`
   - `docs/policy/CP3_READINESS_RUNBOOK.md`

## 12-Month Phases

## Phase F1 (Months 0-2): Required recurring policy gate

Deliverables:

1. Integrate `scripts/run_policy_pipeline.py` into root autonomy cycle.
2. Emit machine-readable policy summary each cycle.
3. Fail closed on strict policy gate regressions.

Exit criteria:

1. Three consecutive cycles with policy pipeline pass.

## Phase F2 (Months 3-5): Security incident readiness

Deliverables:

1. Add policy incident classification matrix (`R0`-`R3`).
2. Add rollback-first runbook for policy regressions.
3. Add incident drill artifacts and replay checks.

Exit criteria:

1. Incident drill reproduces and resolves target regressions deterministically.

## Phase F3 (Months 6-8): Cross-repo policy contracts

Deliverables:

1. Add contract fixtures for AAS/Merlin/Workbench policy envelopes.
2. Enforce compatibility checks in policy pipeline strict mode.
3. Publish weekly policy compatibility score artifact.

Exit criteria:

1. Cross-repo policy contract checks pass in recurring cycle runs.

## Phase F4 (Months 9-12): Autonomous security ratchet

Deliverables:

1. Add policy trend-based ratchet thresholds and alerts.
2. Prioritize policy failures in Guild intake routing.
3. Publish quarterly policy audit bundle with exceptions.

Exit criteria:

1. Zero unresolved critical policy exceptions per quarter.

## KPI targets

1. `policy_pipeline_pass_rate >= 0.95` per month.
2. `open_critical_policy_exceptions = 0` at quarter close.
3. `false_positive_gate_failures` trend decreases quarter over quarter.

## Recurring command baseline

Use this strict command in recurring automation:

```bash
python3 scripts/run_policy_pipeline.py \
  --cycle-id CHIMERA-V2-AUTONOMY-CYCLE-<date> \
  --date <date> \
  --snapshot-id autonomy-cycle-latest \
  --trend-output artifacts/ci/policy_trend_autonomy_cycle_latest.json \
  --status-output docs/policy/CHIMERA_V2_CP4_STATUS_AUTOGENERATED_<date>.md \
  --summary-output artifacts/ci/policy_pipeline_summary.json \
  --strict
```
